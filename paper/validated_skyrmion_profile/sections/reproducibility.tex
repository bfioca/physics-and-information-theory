\section{Reproducibility and certificate provenance}
\label{sec:reproducibility}

The proof is split between human-checkable analytic reductions and a
substantial exact-rational validation code base.  The checker implements
rational interval arithmetic,
rational Taylor or Lagrange enclosures for the required transcendental
functions, interval automatic differentiation, fixed-grid outward rounding,
and exact comparison of the radii-polynomial inequalities.  It does not call a
floating transcendental library in a theorem comparison.

The two hashes printed by \texttt{sha256sum} must be the sharp-snapshot and
radial values displayed above.  The combined AU.1/AU.2 archive also records the
literal command and a dependency ledger.  Its full-file hash is therefore
environment-sensitive.  A byte-identical clean replay used the archived output
paths, Python 3.12.13, NumPy 2.2.6, and no SciPy.

Across supported environments, the audit removes only the top-level provenance
fields \texttt{command} and \nolinkurl{dependency_versions}, then checks the
canonical mathematical-payload digest
\begin{center}
\texttt{f1703ab8f2b3f908a1609d03349c60becdb287225374c91c136fb275053d0f85}.
\end{center}
All exact inputs, source hashes, parameters, and theorem outputs remain inside
that comparison.
The source, certificate archives, manifest, and manuscript are distributed at
\begin{center}
\url{https://github.com/bfioca/physics-and-information-theory}.
\end{center}
The release commit or archival DOI should be cited with any submitted version;
the file-level hashes above identify the present referee snapshot independently
of mutable branch names.
The exact test filenames and a machine-readable manifest are checked as part
of the release audit; no unrelated observer-algebra test is a premise of the
paper.

The union of Python files pinned by the two certificate ledgers and the sharp
tube snapshot is 14,019 physical lines in this repository snapshot, spread
across thirteen modules and audit drivers.  This count deliberately overstates the logical
trusted kernel because it includes proposal generation and presentation code,
but it makes the review surface visible rather than calling it ``small.''
The certificate records a SHA-256 digest for every one of these files.  The
candidate generators may suggest splines and meshes; acceptance depends on the
exact interval validators recomputing joins, residuals, inverse defects, and
closing inequalities.

The archive format records both exact fractions and downward/upward decimal
summaries.  Decimal values printed in the manuscript are presentation layers,
not inputs to subsequent proof steps.  Every composed certificate checks the
identity of its upstream archive before importing a wall slope, inertia,
endpoint jet, or Newton radius.
