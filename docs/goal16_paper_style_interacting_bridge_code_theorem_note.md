# Goal 16: Paper-Style Interacting Bridge Code Theorem

Goal 16 promotes the interacting encoded-bridge benchmark to a theorem-family
artifact. The point is not another example. It is a clean split between a weak
state diagnostic that fails and a stronger Pauli-correlation protocol that
recovers the bridge interaction graph.

## Objects

Fix `m` left mouth qubits, `m` right encoded mouth blocks, a permutation
`pi : L_i -> B_pi(i)`, and a simple graph `G` on the right logical mouths. Each
right block is a stabilizer block of distance `d` with chosen logical
operators `Xbar_j,Zbar_j`; the executable certificate uses independent
`[[5,1,3]]` blocks.

The bridge state is generated by block stabilizers plus Bell-type checks

```text
Z_Li Zbar_pi(i),
X_Li Xbar_pi(i) product_{j in N_G(pi(i))} Zbar_j.
```

Equivalently, start from encoded Bell mouths and apply logical `CZ` gates along
the right-mouth graph `G`.

## Theorem-Style Claims

**Mouth-map recovery.** Full encoded-block mutual information identifies the
paired block `pi(i)` for each left mouth.

**Interaction-graph recovery.** For each paired bridge, solve for the unique
product of right logical `Zbar` operators that makes `X_Li Xbar_pi(i)` a
stabilizer of the physical state. The solution is exactly the neighbor set of
`pi(i)` in `G`. This recovers arbitrary simple graph-`CZ` interactions from
state-derived Pauli correlations.

**Observer algebra.** Conjugating the right logical Pauli algebra by the
recovered `CZ_G` gives the dressed observer algebra. In the representative
three-mouth path witness, the algebra has logical dimension `6`, symplectic
rank `6`, and center dimension `0`.

**Entropy blindness.** For fixed `G`, changing the mouth permutation does not
change labeled physical entropy tables through order `d`. In the executable
`[[5,1,3]]` certificate, aligned and twisted resources match through order `3`
and first split at order `4`.

**Channel transfer.** Removing the inferred interaction and routing by the
inferred mouth map restores exact bridge transfer. Wrong-mouth and mouth-blind
scrambling controls fail.

**Static transition no-go.** The static non-product bridge state still does not
select a north/south screen recovery transition. Opposite screen-channel
completions share the same static entropy, algebra, and bridge-map signature.
An explicit screen dynamics/isometry is the missing ingredient.

## Diagnostic Obstruction

Pairwise inter-bridge mutual information is not an arbitrary-graph reader.
Already for `m=3`, a path/star graph produces pairwise MI excess on all three
pairs, so the weak diagnostic reports the triangle. The Pauli-correlation
reader recovers the actual graph.

This is the theorem-town upgrade over Goal 15: pairwise MI is retained as a
certified failing weaker diagnostic, not used as the arbitrary-graph theorem
input.

## Representative Witness

The default Goal 16 witness uses `m=3`, path graph `((0,1),(1,2))`, and twisted
pairing `(1,0,2)`.

| Diagnostic | Result |
| --- | --- |
| full-block MI mouth map | `(1,0,2)` |
| pairwise MI graph screen | reports triangle `((0,1),(0,2),(1,2))` |
| Pauli-correlation graph reader | recovers path `((0,1),(1,2))` |
| low-order entropy through `d=3` | aligned/twisted match |
| first entropy mismatch | order `4` |
| state-derived transfer | `3` logical qubits |
| mouth-blind scrambling control | `0` structured qubits |

## Bounded Atlas

The executable atlas checks all simple graphs and all mouth permutations for
`m <= 3`, quotienting only by the declared labels.

| `m` | Graphs | Resources | Pauli graph recovery | Pairwise-MI failures |
| --- | ---: | ---: | --- | ---: |
| 1 | 1 | 1 | all pass | 0 |
| 2 | 2 | 4 | all pass | 0 |

It also checks that low-order entropy through `3` is blind to the mouth map for
each graph, state-derived activation has full capacity, and mouth-blind
scrambling has zero structured capacity.

## Claim Boundary

The family proof is finite stabilizer/QEC mathematics. The executable
certificate is bounded to dense checks for `m <= 3`; the Pauli-correlation
argument is the theorem-family proof sketch. This is a finite bridge-channel
benchmark under algebraic ER=EPR motivation, not a continuum wormhole, de
Sitter, dS/CFT, or many-body chaos theorem.
